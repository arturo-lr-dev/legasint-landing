## Introducción

Los despachos de abogados gestionan algunos de los **datos más sensibles** del mundo empresarial: información confidencial de clientes, estrategias legales, documentación privilegiada y datos personales protegidos por GDPR. Sin embargo, muchos despachos todavía operan con medidas de seguridad insuficientes.

En 2025, los **ciberataques a despachos de abogados aumentaron un 47%** según el informe anual de ciberseguridad legal. Los atacantes saben que los bufetes son objetivos valiosos: acceso a información privilegiada, capacidad de pago de rescates, y en muchos casos, medidas de protección deficientes.

Esta guía práctica te ayudará a implementar las medidas esenciales de ciberseguridad en tu despacho, protegiendo tanto a tus clientes como tu reputación profesional.

## 1. Riesgos específicos para despachos

### Ataques comunes al sector legal

**Ransomware:**
- Cifrado de todos los archivos del despacho
- Exigencia de rescate para recuperar acceso
- Paralización completa de la actividad
- **Impacto:** Pérdida económica + reputacional

**Phishing dirigido (Spear Phishing):**
- Emails fraudulentos imitando clientes o colegas
- Solicitudes de transferencias bancarias urgentes
- Robo de credenciales de acceso
- **Caso real 2024:** Despacho español perdió €340,000 por email falso de cliente

**Robo de datos:**
- Acceso no autorizado a expedientes
- Venta de información confidencial
- Espionaje industrial/competencia desleal
- **Consecuencia GDPR:** Multas hasta 20M€ o 4% facturación anual

**Man-in-the-middle (MITM):**
- Interceptación de comunicaciones no cifradas
- Captura de documentos enviados por email
- Robo de credenciales en redes WiFi públicas

### Obligaciones legales

Como despacho de abogados, tienes **obligaciones específicas**:

1. **GDPR (Art. 32):** Medidas técnicas y organizativas apropiadas
2. **Secreto profesional:** Deber de confidencialidad absoluta
3. **Reglamento eIDAS:** Firma electrónica y documentos digitales
4. **Ley Orgánica de Protección de Datos (LOPD):** Normativa española
5. **AI Act (2026):** Si usas herramientas de IA con datos de clientes

**Incumplimiento = Sanciones económicas + Responsabilidad profesional**

## 2. Medidas esenciales de protección

### A. Cifrado de datos

El cifrado es tu primera línea de defensa. Convierte información legible en datos ilegibles sin la clave correcta.

#### Cifrado en disco completo

**Windows:**
```powershell
# Verificar si BitLocker está activo
Get-BitLockerVolume

# Activar BitLocker en unidad C:
Enable-BitLocker -MountPoint "C:" -EncryptionMethod XtsAes256 -UsedSpaceOnly
```

**macOS:** FileVault (activar en Preferencias del Sistema > Seguridad)

**Beneficio:** Si roban el portátil, los datos son inaccesibles sin contraseña.

#### Cifrado de archivos individuales

Para documentos especialmente sensibles:

**7-Zip con AES-256:**
```bash
7z a -p -mhe=on expediente_confidencial.7z carpeta_documentos/
```

**VeraCrypt:** Contenedores cifrados para múltiples archivos

**GPG para emails:**
```bash
# Cifrar archivo antes de enviar
gpg --encrypt --recipient cliente@ejemplo.com documento.pdf
```

#### Cifrado en tránsito

**Email seguro:**
- ✅ Usar TLS (Transport Layer Security) - verificar certificado del servidor
- ✅ S/MIME para emails especialmente confidenciales
- ✅ Evitar enviar información sensible en texto plano

**Transferencia de archivos:**
- ✅ SFTP en lugar de FTP
- ✅ Plataformas con cifrado E2E: Tresorit, Sync.com
- ❌ WeTransfer, Dropbox básico (no E2E)

### B. Copias de seguridad (Backup)

**La regla 3-2-1:**
- **3 copias** de tus datos (original + 2 backups)
- **2 medios diferentes** (disco duro + nube)
- **1 copia offsite** (fuera del despacho)

#### Estrategia de backup recomendada

**Backup diario automático (local):**
```powershell
# Windows Server Backup - script automatizado
$BackupTarget = "D:\Backups"
$SourcePath = "C:\Despacho"

# Crear backup incremental
wbadmin start backup -backupTarget:$BackupTarget -include:$SourcePath -quiet
```

**Backup semanal (nube cifrada):**
- Backblaze B2 con cifrado cliente
- Wasabi (compatible S3, GDPR-compliant)
- Tresorit Business (cifrado E2E, servidores UE)

**Backup mensual (disco externo offline):**
- Disco duro externo guardado en caja fuerte
- Rotación mensual
- **Crítico:** Protege contra ransomware que infecta backups online

#### Pruebas de restauración

**ERROR COMÚN:** Tener backups nunca probados.

```markdown
- [ ] Probar restauración trimestral de archivos aleatorios
- [ ] Documentar tiempo de recuperación (RTO)
- [ ] Verificar integridad de archivos restaurados
- [ ] Entrenar al equipo en proceso de restauración
```

### C. Autenticación de dos factores (2FA)

La **contraseña sola NO es suficiente**. El 80% de brechas de seguridad implican credenciales comprometidas.

#### Implementación 2FA

**Para acceso a sistemas críticos:**
- Email profesional (Microsoft 365, Google Workspace)
- Software de gestión de despachos
- Acceso VPN
- Plataformas de almacenamiento cloud
- Banca online

**Métodos 2FA (del más al menos seguro):**

1. **Hardware keys (FIDO2/U2F):**
   - YubiKey, Google Titan
   - Resistente a phishing
   - **Recomendado para socios y administradores**

2. **Apps de autenticación:**
   - Microsoft Authenticator, Google Authenticator, Authy
   - Genera códigos temporales (TOTP)
   - **Recomendado para todos los empleados**

3. **SMS (evitar si hay alternativas):**
   - Vulnerable a SIM swapping
   - Solo usar si no hay otra opción

**Configuración típica (Microsoft 365):**
```markdown
1. Admin portal > Azure AD > Security > MFA
2. Habilitar MFA para todos los usuarios
3. Permitir: Authenticator app + Hardware token
4. Bloquear: SMS, llamada de voz
5. Requerir re-autenticación cada 30 días
```

### D. VPN (Virtual Private Network)

Obligatorio cuando trabajas fuera de la oficina o en redes no confiables.

#### ¿Cuándo usar VPN?

- ✅ WiFi de hoteles, aeropuertos, cafeterías
- ✅ Teletrabajo desde casa
- ✅ Acceso a sistemas del despacho remotamente
- ✅ Viajes internacionales (especialmente países con censura)

#### Opciones recomendadas

**VPN corporativa (auto-hospedada):**
```bash
# WireGuard - moderno, rápido, seguro
# Servidor en la oficina del despacho
apt install wireguard
wg genkey | tee privatekey | wg pubkey > publickey
```

**VPN comercial (para trabajo remoto):**
- NordVPN Teams (servidores UE, no-logs)
- ProtonVPN Business (Suiza, máxima privacidad)
- Mullvad (anónimo, auditorías públicas)

**Criterios de selección:**
- ✅ Jurisdicción UE o Suiza (GDPR-friendly)
- ✅ Política no-logs verificada
- ✅ Kill switch (bloquea conexión si VPN falla)
- ✅ Cifrado moderno (WireGuard, OpenVPN)
- ❌ Servicios gratuitos (venden tus datos)

## 3. Gestión de contraseñas

### El problema

❌ **Mala práctica común:**
- Misma contraseña en múltiples sitios
- Contraseñas simples: "Despacho2026"
- Anotadas en post-its
- Compartidas por email/WhatsApp

### La solución: Password Manager

**Gestor de contraseñas corporativo:**
- 1Password Business
- Bitwarden Teams (código abierto)
- Keeper Security

**Características esenciales:**
- Generación de contraseñas aleatorias fuertes
- Almacenamiento cifrado (AES-256)
- Compartición segura entre equipo
- Auditoría de contraseñas débiles/reutilizadas
- 2FA integrado

**Ejemplo de política de contraseñas:**
```markdown
Requisitos mínimos:
- Longitud: 16+ caracteres
- Complejidad: Mayúsculas + minúsculas + números + símbolos
- Rotación: Cada 90 días para sistemas críticos
- Prohibido: Palabras del diccionario, datos personales, secuencias
- Único: Nunca reutilizar entre servicios
```

## 4. Protección del correo electrónico

El email es el **vector de ataque principal**. Casi todos los ciberataques empiezan con un email malicioso.

### Medidas anti-phishing

**SPF (Sender Policy Framework):**
```dns
; Autoriza servidores para enviar emails de tu dominio
legasint.com. IN TXT "v=spf1 include:_spf.google.com ~all"
```

**DKIM (DomainKeys Identified Mail):**
```dns
; Firma digital que verifica autenticidad
default._domainkey.legasint.com. IN TXT "v=DKIM1; k=rsa; p=MIGfMA0GCSqGSIb3D..."
```

**DMARC (Domain-based Message Authentication):**
```dns
; Política de autenticación + reportes
_dmarc.legasint.com. IN TXT "v=DMARC1; p=quarantine; rua=mailto:dmarc@legasint.com"
```

### Filtros avanzados

**Microsoft Defender for Office 365:**
- Protección anti-phishing
- Safe Links (análisis de URLs)
- Safe Attachments (sandbox para adjuntos)
- Anti-spoofing inteligente

**Formación del equipo:**
```markdown
Señales de phishing:
- ❌ Remitente desconocido con solicitud urgente
- ❌ Errores gramaticales/ortográficos
- ❌ URLs sospechosas (hovering revela dominio diferente)
- ❌ Solicitudes inusuales (transferencias, cambio contraseña)
- ❌ Adjuntos inesperados (.exe, .zip con contraseña)
```

## 5. Seguridad de red

### Firewall de nueva generación

**Características necesarias:**
- Inspección profunda de paquetes (DPI)
- IPS/IDS (prevención/detección de intrusiones)
- Filtrado por categoría web
- VPN integrada para trabajadores remotos

**Opciones:**
- Fortinet FortiGate (SMB)
- Sophos XG Firewall
- pfSense (open source, auto-hospedado)

### Segmentación de red

**Arquitectura recomendada:**
```
[Internet]
    |
[Firewall]
    |
    ├── [Red Empleados] (VLAN 10) - Acceso completo
    ├── [Red WiFi Invitados] (VLAN 20) - Internet solo
    ├── [Red Servidores] (VLAN 30) - Acceso restringido
    └── [Red IoT/Impresoras] (VLAN 40) - Aislada
```

**Beneficio:** Si comprometen la WiFi de invitados, no acceden a expedientes.

### WiFi segura

**Configuración mínima:**
- ✅ WPA3 (o WPA2-Enterprise si no disponible WPA3)
- ✅ Contraseña fuerte (25+ caracteres aleatorios)
- ✅ Ocultar SSID (opcional, seguridad por oscuridad)
- ✅ Desactivar WPS (vulnerable a ataques)
- ✅ Red invitados separada y aislada

## 6. Protección de endpoints

### Antivirus/EDR de nivel empresarial

**Más allá del antivirus tradicional:**
- EDR (Endpoint Detection and Response)
- Detección basada en comportamiento
- Rollback automático de ransomware

**Soluciones recomendadas:**
- CrowdStrike Falcon
- Microsoft Defender for Endpoint
- SentinelOne
- Bitdefender GravityZone

### Gestión de dispositivos

**MDM (Mobile Device Management) para móviles/tablets:**
- Cifrado obligatorio
- Borrado remoto si se pierde
- Políticas de contraseñas
- Separación datos personales/profesionales

**Actualización de software:**
```powershell
# Windows Update automático (PowerShell como Admin)
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" -Name "NoAutoUpdate" -Value 0

# Forzar instalación de actualizaciones críticas
Install-WindowsUpdate -AcceptAll -AutoReboot
```

**Política de parches:**
- Críticos: Dentro de 48h
- Importantes: Dentro de 7 días
- Opcionales: Evaluación mensual

## 7. Plan de respuesta a incidentes

### Preparación antes del ataque

**Crear un playbook:**
```markdown
# Plan de Respuesta a Incidentes - Despacho Legal

## 1. Detección
- ¿Quién detecta? → Cualquier empleado
- ¿Cómo reporta? → Email a security@despacho.com + llamada al IT Manager

## 2. Contención
- [ ] Aislar dispositivo comprometido de la red (desconectar WiFi/Ethernet)
- [ ] Cambiar contraseñas afectadas (desde dispositivo limpio)
- [ ] Activar modo forense (no apagar, preservar evidencia)

## 3. Análisis
- [ ] Determinar vector de ataque
- [ ] Identificar alcance (¿qué datos accedidos?)
- [ ] Documentar timeline de eventos

## 4. Erradicación
- [ ] Eliminar malware/backdoors
- [ ] Parchear vulnerabilidad explotada
- [ ] Restablecer desde backup limpio si necesario

## 5. Recuperación
- [ ] Restaurar servicios gradualmente
- [ ] Monitorizar actividad anómala
- [ ] Validar integridad de sistemas

## 6. Notificación
- [ ] AEPD (72h si brecha GDPR) → https://www.aepd.es
- [ ] Clientes afectados (sin demora indebida)
- [ ] Seguro de ciberseguridad
- [ ] Colegio de Abogados (según gravedad)

## 7. Post-mortem
- [ ] Análisis de causas raíz
- [ ] Actualización de medidas preventivas
- [ ] Formación al equipo sobre lecciones aprendidas
```

### Contactos de emergencia

**Tener preparado:**
- Empresa de respuesta a incidentes (contratar retainer)
- Perito informático forense
- Abogado especialista en ciberseguridad
- Seguro de ciberriesgos (contacto 24/7)

## 8. Formación y concienciación

### Programa de Security Awareness

**Frecuencia:** Trimestral mínimo

**Temas clave:**
1. Identificación de phishing (con ejemplos reales)
2. Gestión segura de contraseñas
3. Trabajo remoto seguro (VPN, WiFi)
4. Ingeniería social (llamadas fraudulentas)
5. Reporte de incidentes (sin miedo a represalias)

### Simulacros de phishing

**Herramientas:**
- KnowBe4 (simulaciones + formación)
- PhishLabs
- Infosec IQ

**Método:**
```markdown
1. Enviar email simulado de phishing al equipo
2. Trackear quién hace clic / introduce credenciales
3. Formación inmediata personalizada a quienes fallan
4. Repetir mensualmente con escenarios variados
5. Premiar a empleados con mejor track record
```

## 9. Checklist de implementación

### Nivel Básico (esencial para todos)

- [ ] Cifrado de disco completo en todos los portátiles
- [ ] 2FA activado en email y sistemas críticos
- [ ] Gestor de contraseñas corporativo implementado
- [ ] Backup 3-2-1 configurado y probado
- [ ] Antivirus empresarial en todos los endpoints
- [ ] VPN para trabajo remoto
- [ ] Formación básica de ciberseguridad al equipo
- [ ] Plan de respuesta a incidentes documentado

### Nivel Intermedio (recomendado)

- [ ] EDR (Endpoint Detection and Response)
- [ ] SPF + DKIM + DMARC configurados
- [ ] Firewall de nueva generación
- [ ] Segmentación de red (VLANs)
- [ ] MDM para dispositivos móviles
- [ ] Simulacros de phishing trimestrales
- [ ] Auditoría de seguridad anual por tercero
- [ ] Seguro de ciberriesgos contratado

### Nivel Avanzado (para despachos grandes)

- [ ] SOC (Security Operations Center) o MSSP
- [ ] SIEM (Security Information and Event Management)
- [ ] Zero Trust Architecture
- [ ] Pentesting anual
- [ ] Bug Bounty program
- [ ] DLP (Data Loss Prevention)
- [ ] Certificación ISO 27001
- [ ] Red Team exercises

## Conclusión

La ciberseguridad no es opcional para despachos de abogados modernos. Es una **obligación legal, ética y profesional**. Los clientes confían en ti sus secretos más valiosos; protegerlos es parte fundamental de tu deber de confidencialidad.

**Tres principios fundamentales:**

1. **Defensa en capas:** No existe la "bala de plata". Combina múltiples medidas.
2. **El eslabón más débil:** Tu seguridad es tan fuerte como tu empleado menos formado.
3. **Mejora continua:** Las amenazas evolucionan; tus defensas también deben hacerlo.

**Próximos pasos:**
1. Completa el checklist de nivel básico este mes
2. Programa auditoría de seguridad externa
3. Implementa formación trimestral obligatoria
4. Revisa y actualiza tu plan de respuesta a incidentes

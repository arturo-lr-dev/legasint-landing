## Introducción

React ha evolucionado significativamente desde su lanzamiento. En este artículo, exploraremos las mejores prácticas que todo desarrollador de React debería conocer en 2026.

## 1. Componentes Funcionales

Los componentes funcionales con hooks son ahora el estándar:

```tsx
// ✅ Recomendado
const UserProfile: React.FC<{ name: string }> = ({ name }) => {
  const [isActive, setIsActive] = useState(false);

  return (
    <div className={isActive ? 'active' : ''}>
      <h1>{name}</h1>
    </div>
  );
};
```

## 2. Custom Hooks

Extrae la lógica reutilizable en custom hooks:

```tsx
function useLocalStorage<T>(key: string, initialValue: T) {
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  const setValue = (value: T) => {
    setStoredValue(value);
    window.localStorage.setItem(key, JSON.stringify(value));
  };

  return [storedValue, setValue] as const;
}
```

## 3. Manejo de Estado

Utiliza el estado de forma inteligente:

- **useState**: Para estado local simple
- **useReducer**: Para estado complejo con múltiples sub-valores
- **Context**: Para estado global ligero
- **Zustand/Jotai**: Para estado global más complejo

## 4. Optimización de Rendimiento

### Memoización

```tsx
// Componente memoizado
const ExpensiveComponent = React.memo(({ data }) => {
  return <div>{/* ... */}</div>
});

// Valores memoizados
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);

// Callbacks memoizados
const memoizedCallback = useCallback(() => {
  doSomething(a, b);
}, [a, b]);
```

## 5. TypeScript

> Siempre usa TypeScript para aplicaciones en producción

Define interfaces claras para tus props y estado:

```tsx
interface User {
  id: string;
  name: string;
  email: string;
}

interface UserListProps {
  users: User[];
  onSelect: (user: User) => void;
}
```

Seguir estas prácticas te ayudará a crear aplicaciones React más mantenibles, escalables y eficientes. Recuerda que el código limpio es código que otros (y tu yo del futuro) pueden entender fácilmente.

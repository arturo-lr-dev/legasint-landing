## Introducción

La **inteligencia artificial** está transformando la práctica legal en España y en toda la Unión Europea. Mientras que el **EU AI Act** establece el marco regulatorio obligatorio, los despachos de abogados necesitan un **marco ético robusto** para guiar el uso responsable de estas tecnologías.

El 2026 marca un punto de inflexión: la implementación completa del AI Act y la adopción masiva de herramientas de IA en el sector legal. Esta guía proporciona un marco ético práctico para despachos españoles que desean liderar en la **adopción responsable de IA**.

## 1. Principios Éticos Fundamentales

### 1.1 Transparencia Total

Los clientes tienen derecho a saber cuándo se utiliza IA en sus asuntos:

```
✅ Buenas prácticas:
- Informar al cliente sobre el uso de IA en el asunto
- Documentar las herramientas IA utilizadas
- Mantener registros de decisiones algorítmicas

❌ A evitar:
- Uso oculto de IA sin consentimiento informado
- Presentar resultados IA como trabajo humano exclusivo
- Ocultar las limitaciones de las herramientas utilizadas
```

### 1.2 Responsabilidad Humana

La **supervisión humana** es un principio irrenunciable:

- **Revisión final**: Todo trabajo IA debe ser revisado por abogados cualificados
- **Asumir responsabilidad**: El abogado es responsable de todos los resultados, independientemente de la fuente
- **Capacidad de apelación**: Los clientes deben poder cuestionar decisiones algorítmicas

### 1.3 Justicia y Equidad

Las herramientas de IA no pueden amplificar sesgos existentes:

- **Auditoría de sesgos**: Evaluar regularmente las herramientas para detectar sesgos de género, raza o socioeconómicos
- **Diversidad de datos**: Garantizar que los sets de entrenamiento representen la diversidad de la sociedad española
- **Acceso equitativo**: No utilizar IA para discriminar en servicios legales

## 3. Categorización Ética de Uso de IA

### 3.1 Uso Aceptable (Bajo Riesgo)

Estas aplicaciones se consideran **éticamente aceptables**:

| Aplicación | Riesgo Ético | Medidas |
|-----------|-------------|---------|
| Búsqueda jurídica automatizada | Bajo | Verificación de fuentes primarias |
| Redacción de borradores | Bajo | Revisión humana exhaustiva |
| Traducción de documentos | Bajo | Revisión por abogado bilingüe |
| Organización de evidencia | Medio | Validación de contexto |

### 3.2 Uso Requiere Supervisión (Riesgo Medio)

**Requiere supervisión humana activa**:

```typescript
// Ejemplo: análisis de contratos con IA
function analisisContratoIA(contrato) {
  const resultadoIA = analizarConIA(contrato);

  // ✅ Supervisión humana OBLIGATORIA
  const revisionHumana = revisarAbogado(resultadoIA);
  
  if (revisionHumana.riesgos.length > 0) {
    return "Requiere revisión adicional";
  }

  return "Aprobado con reservas";
}
```

Aplicaciones de riesgo medio:
- **Análisis predictivo** de resultados judiciales
- **Clasificación automática** de documentos legales
- **Extracción de información** de contratos complejos
- **Asistentes virtuales** para atención al cliente

### 3.3 Uso Restringido (Alto Riesgo)

**Prohibido o altamente regulado** según el AI Act:

- ⚠️ **Decisiones judiciales automatizadas** (Art. 5 AI Act)
- ⚠️ **Evaluación de credibilidad** de personas
- ⚠️ **Perfiles biométricos** para identificación legal
- ⚠️ **Análisis emocional** en negociaciones

> **Nota**: El AI Artículo 5 prohíbe ciertas prácticas de IA consideradas inaceptables, incluyendo la toma de decisiones judiciales sin supervisión humana.

## 4. Evaluación Ética de Herramientas IA

### 4.1 Checklist de Evaluación

```markdown
Antes de adoptar una herramienta IA, verificar:

✅ Transparencia del proveedor
   - ¿Documentan el funcionamiento del modelo?
   - ¿Proveen información sobre datos de entrenamiento?

✅ Responsabilidad
   - ¿Aceptan el proveedor responsabilidad por errores?
   - ¿Incluye el DPA cláusulas de indemnización?

✅ Cumplimiento regulatorio
   - ¿Certificación AI Act (cuando aplique)?
   - ¿Cumple GDPR en protección de datos?

✅ Ética y derechos humanos
   - ¿Política de derechos humanos del proveedor?
   - ¿Mecanismos de apelación disponibles?

✅ Sostenibilidad
   - ¿Impacto ambiental del modelo?
   - ¿Política de uso sostenible de recursos?
```

```

   ↓

   ↓
```

## 6. Casos Prácticos Éticos

### Caso 1: IA en Due Diligence M&A

**Situación**: Un despacho utiliza IA para analizar 5,000 documentos en una operación M&A.

**Enfoque ético**:
```python
def due_diligence_etica(documentos, herramienta_ia):
    # 1. Informar al cliente
    informar_cliente("Usaremos IA para análisis preliminar")

    # 2. Establecer expectativas
    expectativas = {
        "cobertura": "95% documentos",
        "revision_humana": "100% hallazgos relevantes",
        "limitaciones": ["Contexto complejo requiere revisión manual"]
    }

    # 3. Ejecutar con supervisión
    resultados = herramienta_ia.analizar(documentos)
    hallazgos = filtrar_riesgos(resultados, umbral="medio")

    # 4. Revisión humana obligatoria
    hallazgos_validados = []
    for hallazgo in hallazgos:
        validado = abogado_revisa(hallazgo)
        hallazgos_validados.append(validado)

    # 5. Documentar proceso
    documentar_proceso({
        "herramienta": herramienta_ia.nombre,
        "resultados_ia": len(resultados),
        "revision_humana": len(hallazgos_validados),
        "tiempo_ahorrado": "3 días"
    })

    return hallazgos_validados
```

### Caso 2: IA en Asesoría Jurídica

**Situación**: Un abogado utiliza ChatGPT para generar borradores de opinión legal.

**Principios éticos**:
1. **No presentar como trabajo propio exclusivo**: Atribuir uso de IA
2. **Verificar jurisprudencia**: Las herramientas pueden alucinar casos
3. **Añadir valor humano**: El abogado debe aportar análisis, no solo formatear

```
Proceso recomendado:

Paso 1: Consultar fuentes primarias (BOE, jurisprudencia)
Paso 2: Usar IA para estructura del borrador
Paso 3: Verificar cada afirmación con fuentes
Paso 4: Añadir análisis personal y experiencia
Paso 5: Finalizar con revisión jurídica rigurosa
```
